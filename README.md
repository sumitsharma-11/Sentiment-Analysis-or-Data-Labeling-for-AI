# Sentiment-Analysis-or-Data-Labeling-for-AI

Human-in-the-Loop AI: Sentiment Analysis & Data Labeling Audit 🤖🤝🧑‍💻

Project Overview
This project demonstrates a full-cycle data analysis and machine learning workflow designed to improve the quality of AI-labeled datasets. It uses Natural Language Processing (NLP) to analyze customer sentiment and identifies high-risk or ambiguous entries that require human auditing.
By combining automated AI scoring with a Human-in-the-Loop (HITL) framework, this project ensures the creation of high-quality "Ground Truth" data essential for retraining more accurate AI models.

Key Features
Automated Sentiment Scoring: Utilizes NLTK's VADER and Logistic Regression to classify text data (Amazon Reviews) into sentiment categories.
AI Audit Trigger: Implements logic to flag entries where the AI's confidence is low or where specific "High Risk" indicators are detected.
Human-in-the-Loop (HITL) Simulation: A Python-based workflow for manual verification of AI labels, generating a verified dataset for model improvement.
Interactive Power BI Dashboard: A comprehensive visualization of audit progress, risk distribution, and AI performance metrics.

Tech Stack
Python (Jupyter Notebook): Data processing, NLP, and audit simulation.
Libraries: pandas, nltk, scikit-learn.
Power BI: Interactive visualization and reporting of audit results.
NLP Models: VADER (Sentiment Intensity Analyzer) and TF-IDF Vectorization.

Repository Contents
PythonScript.ipynb: The core processing engine that performs sentiment analysis and runs the human-in-the-loop audit simulation.
Sentiment Analysis or Data Labeling for AI.pbix: The Power BI dashboard for monitoring AI health and audit status.
amazon_reviews.csv: Sample dataset used for analysis (not included in this preview, but used as source).
peroptyx_ground_truth.csv: The final "verified" dataset generated by the human-in-the-loop process.

Dashboard Insights (Power BI)
The dashboard provides a deep dive into the following metrics:
Average AI Score: Tracking the overall sentiment trend across the dataset.
High-Risk Flags: Identifying which entries the AI is struggling to classify.
Audit Reasons: Breakdown of why specific records were flagged for human review (e.g., sarcasm, ambiguous language, or conflicting scores).
Decomposition Tree: Exploring the relationship between profile names, sentiment scores, and audit flags.

Project Workflow
Ingestion: Loading raw review data.
AI Labeling: Running NLP models to assign sentiment scores and reasons.
Flagging: Filtering records that meet the "Audit Required" criteria.
Verification: Manual (simulated) human review to correct AI errors.
Visualization: Pushing the verified data to Power BI to monitor the delta between AI and Human labels.
